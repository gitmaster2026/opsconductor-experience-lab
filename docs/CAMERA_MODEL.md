# Camera Model

The camera defines how users move through the operational workspace.

## Principle

Navigation should feel like movement through operational understanding, not page changes.

## Controls

V4 controls:

- zoom slider
- mouse wheel zoom
- optional pan/drag in Universe

## Zoom hierarchy

Zoom should eventually reveal deeper operational context:

1. Organization
2. Site / Plant
3. Customer
4. Program
5. Commitment
6. Operational Object
7. Evidence
8. Source Record

V4 can approximate this by changing detail density and label visibility.

## Focus behavior

Selecting an object may move the camera toward that object.

Switching lenses must preserve selected object/focused commitment.

## Separation from time

Zoom never changes time.

Time never changes zoom unless explicitly designed as a cinematic replay in a future version.
