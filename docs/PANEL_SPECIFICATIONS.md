# Panel Specifications

Panels are persistent context surfaces.

They are not pages.

## V1-A panel rule

Panels must preserve the selected NorthRiver investigation thread while the user moves deeper through:

Executive Signal -> Commitment -> Demand -> Shortage -> Recommendation -> Decision -> Evidence -> Operational Relationships -> Timeline -> Source Records

The Passport is the universal selected-object experience. Dashboard, Evidence, Timeline, Source Records, and Jarvis should reinforce the same selected object rather than creating separate stories.

## Left Context Panel

The left panel has two primary modes.

### Dashboard mode

Purpose: answer what deserves attention now.

Required behavior:

- show operational health
- show revenue at risk
- show commitments at risk
- show active recommendations
- show top commitment risks
- all KPI/risk cards are clickable
- clicking updates selected object/focused commitment and may switch left panel to Passport
- the flagship executive signal should resolve to `RB-CPP-HORIZON`
- no Meridian, alternate customer, or independent demo storyline should appear

### Passport mode

Purpose: show the biography of the selected operational object.

Required sections:

- Overview
- Current Risk
- Relationships
- Recommendations
- Evidence
- Timeline / Operational History
- Source Records

Passport must update with time.

Evidence and recommendations should only appear when visible in the selected time slice.

Passport must support honest depth:

- flagship Horizon CPP-1000 objects may show full narrative depth
- non-flagship NorthRiver shortage rows may show real coverage/recommendation data plus a gated deeper-investigation state
- no panel should fabricate an approval, notification, export, collection, or automation workflow

## Evidence panel behavior

Evidence is not decorative. It must cite a governed fixture object:

- shortage recommendation id
- operational domain object id
- source table
- source record id / source identifier

If evidence is not available for a selected object, show an honest unavailable state.

## Timeline panel behavior

Timeline must be a synchronized view of the selected investigation and global time slice.

It should reveal the same NorthRiver operational chain rather than resetting to a generic activity feed.

## Source Records behavior

Source Records are the terminal proof layer of the V1-A story.

For the flagship investigation, `CESC-NR-2026-014` is the current terminal source record. Other records may appear as supporting source records where they map to the static fixture.

## Right Jarvis Panel

Purpose: persistent deterministic intelligence.

Jarvis is not a chatbot in the lab.

Required behavior:

- summarize the selected object
- summarize the active workspace lens
- summarize the active time slice
- identify important related objects
- suggest deterministic next steps
- cite evidence/source record IDs when visible
- call out gated states honestly when decision depth or full narrative depth is unavailable

Jarvis must never invent fields or facts outside the static data snapshot.
